# Name: wgs_metrics

# Description
    Runs whole-genome QC metrics for each BAM using Picard-based metrics rules
    defined in `wgs_metrics.mk`.

    The workflow produces per-sample and summary tables for:
    - BAM index stats
    - alignment summary metrics
    - insert-size metrics
    - OxoG metrics
    - GC-bias metrics
    - whole-genome coverage metrics
    - duplicate metrics

# Inputs
    - `bam/<sample>.bam`
    - Picard / Java configuration from `config.inc` and `wgs_metrics.mk`
    - `scripts/wgs_metrics.R` for summary-table generation

# Outputs
```
metrics/<sample>.idx_stats.txt
metrics/<sample>.aln_metrics.txt
metrics/<sample>.insert_metrics.txt
metrics/<sample>.oxog_metrics.txt
metrics/<sample>.gc_metrics.txt
metrics/<sample>.gc_metrics_summary.txt
metrics/<sample>.wgs_metrics.txt
metrics/<sample>.duplicate_metrics.txt

summary/idx_metrics.txt
summary/aln_metrics.txt
summary/insert_metrics.txt
summary/oxog_metrics.txt
summary/gc_metrics.txt
summary/wgs_metrics.txt
summary/duplicate_metrics.txt
```

# Duplicate-metrics behavior
- Duplicate metrics are **enabled by default**.
- If alignment already produced `metrics/<sample>.dup_metrics.txt`, the WGS QC
  workflow reuses that file and copies it to `metrics/<sample>.duplicate_metrics.txt`.
- If no prior duplicate-metrics file is available, `wgs_metrics.mk` falls back
  to a standalone Picard `MarkDuplicates` metrics pass.
- To disable duplicate metrics for a project:

```
make wgs_metrics ENABLE_DUPLICATE_METRICS=false
```

# Usage
```
make wgs_metrics USE_CLUSTER=false
```

    Common overrides:

```
make wgs_metrics METRICS_PICARD_JAR=/path/to/picard.jar
make wgs_metrics ENABLE_OXOG_METRICS=false
make wgs_metrics DUPLICATE_METRICS_WALLTIME=24:00:00
```

# Notes
- This workflow is intended for WGS and does not compute hybrid-selection style
  metrics used by targeted/exome pipelines.
- The insert-size metrics step uses a lower `MINIMUM_PCT` threshold (`0.05`)
  than the standard BAM metrics workflow, which is more appropriate for WGS.
- Summary tables are generated by `scripts/wgs_metrics.R`, which now tolerates
  missing columns across samples.
